# Human-AI Text Checker

A full-stack application for detecting whether text was written by a human or generated by AI. Uses LangChain RAG (Retrieval-Augmented Generation) for enhanced analysis accuracy.

## Architecture

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────────────┐
│   React UI      │────▶│  Node.js API    │────▶│  FastAPI Inference      │
│   (Port 3000)   │     │  (Port 3001)    │     │  (Port 8000)            │
│                 │     │                 │     │                         │
│  - User Interface│     │  - Routing      │     │  - LangChain RAG        │
│  - Analysis View │     │  - Validation   │     │  - ChromaDB Vectors     │
│  - History      │     │  - History      │     │  - Llama API Inference  │
└─────────────────┘     └─────────────────┘     └─────────────────────────┘
```

### Components

1. **React UI (`/ui`)** - Frontend application
   - Modern React interface for text analysis
   - Displays AI vs Human probability with visualizations
   - Analysis history management

2. **Node.js API (`/api`)** - Main API server
   - Express.js REST API
   - Request validation and rate limiting
   - Analysis history storage
   - Proxies inference requests to FastAPI

3. **FastAPI Inference (`/server`)** - ML Inference server
   - LangChain RAG system with ChromaDB
   - HuggingFace embeddings (all-MiniLM-L6-v2)
   - Llama API integration for text analysis
   - Knowledge base for AI detection patterns

## Quick Start

### Prerequisites

- Node.js 18+
- Python 3.9+
- Llama API key (or compatible API)

### 1. Setup FastAPI Inference Server

```bash
cd server

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your LLAMA_API_KEY

# Start server
uvicorn main:app --reload --port 8000
```

### 2. Setup Node.js API Server

```bash
cd api

# Install dependencies
npm install

# Configure environment (already set up)
# Edit .env if needed

# Start server
npm run dev
```

### 3. Setup React Frontend

```bash
cd ui

# Install dependencies
npm install

# Start development server
npm start
```

### 4. Access the Application

- **Frontend**: http://localhost:3000
- **Node.js API**: http://localhost:3001
- **Inference API**: http://localhost:8000

## API Endpoints

### Node.js API (Port 3001)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/analysis/analyze` | POST | Analyze text (standard) |
| `/api/analysis/langchain` | POST | Analyze with LangChain RAG |
| `/api/analysis/history` | GET | Get analysis history |
| `/api/analysis/:id` | GET | Get specific analysis |
| `/api/analysis/:id` | DELETE | Delete analysis |
| `/api/health` | GET | System health check |
| `/api/health/inference` | GET | Inference server status |
| `/api/health/rag` | GET | RAG system status |

### FastAPI Inference (Port 8000)

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/analyze` | POST | Standard analysis with RAG |
| `/analyze/langchain` | POST | Full LangChain RAG analysis |
| `/rag/status` | GET | RAG system statistics |
| `/rag/test` | GET | Test RAG retrieval |
| `/health` | GET | Inference health check |

## LangChain Implementation

The LangChain RAG system provides:

1. **Vector Store (ChromaDB)**
   - Persistent storage for AI detection knowledge
   - Semantic similarity search
   - Efficient document retrieval

2. **Embeddings (HuggingFace)**
   - Model: `all-MiniLM-L6-v2`
   - Normalized embeddings for better similarity matching
   - CPU-optimized inference

3. **LLM Chain**
   - Structured prompt templates
   - Context-aware analysis
   - JSON-formatted output

4. **Knowledge Base**
   - AI writing patterns (repetition, consistency, verbosity)
   - Human writing patterns (variation, emotion, specificity)
   - Grammar and style indicators
   - Flow and structure patterns

```

```

```

NODE_ENV=development
FRONTEND_URL=http://localhost:3000
INFERENCE_URL=http://localhost:8000
```

### UI (.env)
```
REACT_APP_API_URL=http://localhost:3001
```
